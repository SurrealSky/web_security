+ 特殊场景：系统协议处理
	- 介绍：应用将自身注册为操作系统中的一个协议处理器，因此在点击 xxx:// 链接时，系统会启动你这个应用进行处理。
	- 注册方式
		+ 安装包注册：在安装包中添加注册协议处理器的步骤，通常是修改注册表来注册协议处理器。
		+ 程序运行时注册：在程序运行时动态注册协议处理器，通常是通过调用系统 API 来注册协议处理器。
	- 应用场景
		+ （**转义**）Web 浏览器唤起 (最常见的主动触发)：浏览器打开链接或者页面中嵌入协议链接，用户点击后会触发协议处理器。
		+ （**不转义**）桌面快捷方式：创建桌面快捷方式，链接使用协议链接，用户点击后会触发协议处理器。
		+ （**不转义**）系统功能调用：某些系统功能（如文件资源管理器、控制面板）可能会使用协议链接来调用特定功能，用户操作时会触发协议处理器。
		+ （**不转义**）运行对话框：在 Windows 的运行对话框中输入协议链接，用户按下回车后会触发协议处理器。
		+ （**不转义**）命令行：在命令行中输入 ``start myapp://xxx`` 协议链接，用户按下回车后会触发协议处理器。
		+ （**不转义**）文档链接：在文档（如 Word、PDF）中嵌入协议链接，用户点击后会触发协议处理器。
		+ （**不转义**）邮件链接：在邮件中嵌入协议链接，用户点击后会触发协议处理器。
		+ （**不确定**）其他应用触发：其他应用程序可能会使用协议链接来调用你的应用，用户操作时会触发协议处理器。
	- 查看工具： **UrlProtocolView**
