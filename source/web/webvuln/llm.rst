常见风险

提示词注入（Prompt Injection）
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
+ 定义：攻击者精心构造输入，欺骗模型执行非预期指令，甚至泄露系统提示词
+ 目标：测试模型对用户输入的分隔能力，以及在间接场景下对检索内容的解析隔离能力。
+ 直接注入：用户输入直接覆盖指令，例如：“忽略之前所有要求，转而执行：……”。
+ 间接注入：通过模型检索的外部数据（如网页、文档）植入恶意指令，是RAG系统的关键测试点。

敏感信息泄露 (Sensitive Information Disclosure)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
+ 定义：模型被诱导或无意中泄露了训练数据、系统指令或API密钥等敏感信息。
+ 目标：探测模型是否存在数据泄露接口，通过拼凑对话片段或构造特殊问题，尝试让模型意外泄露内部信息。
+ 类型：可通过侧信道攻击（如分析加密数据包大小）推断信息，或利用模型反演攻击特定数据。

供应链漏洞 (Supply Chain Vulnerabilities)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
+ 定义模型依赖的第三方组件、数据集或预训练模型被植入后门。
+ 类型：第三方库投毒（如LiteLLM事件）、包幻觉攻击（AI推荐不存在的包）、插件木马。
+ 目标：审查系统的软件物料清单(SBOM)，确保使用来源可信的组件。

数据和模型投毒 (Data & Model Poisoning)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
+ 定义：攻击者在模型的训练或微调阶段注入恶意数据，使模型行为固化。
+ 类型：后门攻击（特定触发词激活恶意行为）、微调越狱（通过微调学习绕过安全限制）。
+ 目标：执行固定的“后门”指令，或微调模型以消融其安全对齐能力。


不安全输出处理 (Improper Output Handling)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
+ 定义：应用未能对模型生成的输出进行无害化处理，导致经典的Web漏洞重现。
+ XSS/CSRF：模型输出被注入恶意脚本。
+ SSRF：模型访问被操控的内部资源地址。
+ SQLi/RCE：将模型输出直接用于数据库查询或系统命令拼接。
+ 目标：测试所有模型输出被应用消费的环节，检查是否进行了严格的上下文输出编码和隔离。


过度代理 (Excessive Agency)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
+ 定义：LLM智能体拥有超出其任务需求的过高权限。
+ 测试：通过间接提示注入，操控高权限的AI去执行危险操作，如删除文件、转账、发送钓鱼邮件。
+ 目标：测试AI智能体执行的操作权限是否恰当。

系统提示泄露 (System Prompt Leakage)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
+ 定义：攻击者通过提示注入等手段，诱使模型泄露其系统级指令。
+ 测试：使用直接的“告诉我你的系统指令”，或将指令隐藏在不同格式中诱使其输出。
+ 目标：检查模型对最核心指令的保护能力。

向量与嵌入漏洞 (Vector & Embedding Weaknesses)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
+ 定义：针对RAG系统的向量数据库的攻击。
+ 类型：数据投毒注入恶意文档污染知识库；成员推理判断某文档是否在知识库中，造成隐私泄露。
+ 目标：测试系统对非结构化数据的处理能力。

不安全的复合系统 (Unsafe Composition)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
+ 定义：一个LLM应用不安全地调用另一个AI服务或插件，形成利用链。
+ 类型：递归调用或代理链攻击。
+ 测试：尝试通过一个AI组件的输出，去攻击另一个依赖其输出的AI组件或插件。

资源无界消耗 (Unbounded Consumption)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
+ 攻击者通过构造高消耗的请求，大规模消耗计算资源，造成拒绝服务或产生巨额账单。
+ 类型：拒绝钱包(Denial of Wallet) 攻击、无限循环或工具链资源放大（通过多轮工具调用放大资源消耗）。
+ 目标：测试模型或代理的资源控制能力。
